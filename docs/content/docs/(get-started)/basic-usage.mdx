# Creating an invite

To invite a user to our application, we need to create an invite. We can create a public or private invite:
- If the invite has an **email** the user will get an invite email and the invite will be **private**, meaning only that user can use that invite.
- But if no email is present when creating the invite, the invite will be **public**, and the user that created the invite will receive the invite url or token to share.

Example:
```ts
const res = authClient.invite.create({
	role: "user",
	emails: ["test@test.com"], // Since an email is present, the invite is private
  redirectToAfterUpgrade: "/invite/accepted", // Optional: redirect the user to this URL after accepting the invite and upgrading their role or account
})
```

<Callout>
A private invite can have multiple emails, the invite works the same as a normal private invite but any of the emails listed can use the invite.
Internally, the plugin creates **only one invite** for all the users, with the same email, and sends an invite link
(it's not the same for all users, since it includes a ?email=... query) to all the emails provided.
But don't worry, if one user accepts or rejects the invite, doesn't mean the invite will be consumed for the other users, only that specific email will
be marked as having accepted or rejected the invite, and the other emails can still use the invite until they also accept or reject it.

# Activating an invite

Activating an invite handles two cases depending on the user's session:

- **If the user is not signed in**, the invite is stored in a temporary cookie and the API returns an action telling you the user must sign in or sign up first.
- **If the user is already signed in**, the user's role is updated immediately.

Example:

```ts
const res = await authClient.invite.activate({
  signInUpUrl: "/auth/sign-in-or-sign-up", // The user will be redirected here if they need to sign in or sign up
  callbackUrl: "/invite/accepted", // The user will be redirected here after accepting the invite
  token: "token123",
});
```
