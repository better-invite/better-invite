# Creating an invite

To invite a user to our application, we need to create an invite. We can create a public or private invite:
- If the invite has an **email** the user will get an invite email and the invite will be **private**, meaning only that user can use that invite.
- But if no email is present when creating the invite, the invite will be **public**, and the user that created the invite will receive the invite url or token to share.

Example:
```ts
const res = await authClient.invite.create({
	role: "user",
	email: ["test@test.com"], // Since an email is present, the invite is private
  redirectToAfterUpgrade: "/invite/accepted", // Optional: redirect the user to this URL after accepting the invite and upgrading their role or account
})
```

<Callout>
A private invite can include multiple emails. Internally, the plugin creates **one shared invitation** (one token) that stores the full recipient list, and sends each recipient their own invite link
(links differ because they include a `?email=...` query).
When one recipient accepts or rejects, only that email is removed from the invite; the others can still use it.
For multi-email invites, Better Invite defaults to one use per recipient (`maxUsesPerUser: 1`) so the first acceptance does not exhaust the shared invite for everyone else.
</Callout>

If the invite is private, the plugin will send an email to the user (or users) with the invite link (see [Accept Invite Callback](/docs/core/api#accept-invite-callback)).

# Accepting an invite

Accepting an invite depends on how the user arrives:

- **Email / browser link** → [Accept Invite Callback](/docs/core/api#accept-invite-callback) (`GET /invite/:token`). If the user is not signed in, the invite token is stored in a cookie and they are redirected to sign in or sign up.
- **Authenticated app UI** → [Accept Invite](/docs/core/api#accept-invite) (`POST /invite/accept`, requires a session). The user's role is updated immediately.

Example (authenticated accept):

```ts
const res = await authClient.invite.accept({
  signInUpUrl: "/auth/sign-in-or-sign-up", // Used when calling the callback flow without a session
  callbackUrl: "/invite/accepted", // The user will be redirected here after accepting the invite
  token: "token123",
});
```
