# Custom activate invite

Instead of relying on `/api/invite/activate`, you can fully control the flow using your own UI.

The idea is simple:

- In `sendInvite`, generate a custom URL using the token
- Handle everything inside `/invite/[token]`
- Let the user accept or reject
- Act based on the response

---

<Steps>

<Step>

## Generate the invite link

Use the option `defaultCustomInviteUrl` or `customInviteUrl` to generate a custom URL that includes the token.

<Callout>
`{token}`, `{email}` and `{callbackUrl}` will be replaced with their respective values.
</Callout>

```ts
defaultCustomInviteUrl: '/invite/{token}?email={email}?callbackUrl={callbackUrl}'
```

In `sendInvite`, simply send the custom url to the user:

```ts
async sendUserInvitation({ email, role, url }) {
  await sendInvitationEmail(role, email, url);
}
```

This completely replaces the default url (`/api/invite/activate`).
</Step>

<Step>
## Create `/invite/[token]` page

This page is where you control the full experience.

You can render a simple UI with:

- **Accept invite**
- **Reject invite**
</Step>

<Step>
## Check for user session

When the page loads, check if the user is already authenticated:

```ts
const { user } = authClient.getSession();
```

- If the user is not authenticated, you redirect them to sign in/up, with a callback url that points back to the same page (`/invite/[token]`).
- If the user is authenticated, you can show the accept/reject buttons.

<Callout title="Tip">
When redirecting the user to the sign in/up page, you can include a custom email query param
to prefill the email field, make sure to implement the login in your UI to support this.
</Callout>

</Step>

<Step>
## Accept the invite

Now that the user is authenticated and clicks **Accept** you can call the activation endpoint:

```ts
const res = await authClient.invite.activate({ token });
```

</Step>

<Step>
## Reject the invite

If the user clicks **Reject**:

```ts
await authClient.invite.reject({ token });
```
</Step>

<Step>

## Redirect to after upgrade

Now that the user has accepted or rejected the invite, you can redirect them to the appropriate page. Use `callbackUrl` in activate/reject to specify where to redirect the user after accepting or rejecting the invite.

</Step>

<Step>
## Summary

- Generate link: `/invite/${token}`
- Show your custom UI
- If accepted:
  - call `activate()`
  - if auth is required → redirect using `redirectTo`
- If rejected:
  - call `reject()`
  - redirect manually
</Step>
</Steps>

---

## Notes

- You do NOT need to manually resume the flow after login
- The plugin automatically continues after authentication
- This approach is ideal for:
  - better onboarding UX
  - full control over the flow
  - role or membership-based apps
